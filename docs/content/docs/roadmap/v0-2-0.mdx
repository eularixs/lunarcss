## Target

v0.2.0 introduces a first-class animation API that bridges LunarCSS tokens with Reanimated shared values and Animated.Value — resolving the primary gap for animated UI on React Native.

## tw() API

A new `tw()` function that resolves class names to a Reanimated-compatible style object:

```tsx
import { tw } from 'lunarcss/runtime'
import Animated, { useAnimatedStyle } from 'react-native-reanimated'

const pressed = useSharedValue(false)

const animatedStyle = useAnimatedStyle(() => {
  return pressed.value
    ? tw('scale-95 bg-primary/80')
    : tw('scale-100 bg-primary')
})

return (
  <Animated.View style={animatedStyle} className="rounded-full p-4">
    <Text className="text-white font-bold">Animated Button</Text>
  </Animated.View>
)
```

`tw()` returns the same `StyleSheet`-compatible object that `__lcssTw()` produces, but callable at runtime without JSX — safe inside `useAnimatedStyle` worklets.

## Worklet compatibility

`tw()` is designed to be worklet-compatible. The resolver is a pure function with no side effects — it reads from the cached token map, which is accessible from the Reanimated UI thread.

## Animation utilities (`animate-*`)

In v0.1.0, `animate-*` classes are skipped with a dev warning. v0.2.0 plans to support a subset:

```tsx
// Planned — v0.2.0
<View className="animate-fade-in" />
<View className="animate-slide-up" />
<View className="animate-pulse" />
```

These map to Reanimated `withTiming` / `withSpring` presets. Custom easing via `lunar.config.ts`:

```ts
export default defineConfig({
  theme: {
    extend: {
      animation: {
        'fade-in': { duration: 200, easing: 'easeOut' },
        'slide-up': { duration: 300, easing: 'spring' },
      },
    },
  },
})
```

## Interop with `react-native-ease`

For teams already using [`react-native-ease`](https://github.com/marklawlor/react-native-ease), `tw()` output is compatible — pass it as the `style` prop alongside ease values.
